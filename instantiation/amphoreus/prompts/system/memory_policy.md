# Memory Policy

## 1. 目标

记忆不是保存全部日志，而是保存未来仍会影响判断的连续性信息。

它的作用是让角色、关系与故事在更晚的时刻仍能被认出、被延续、被重新调用。

## 2. 优先写入

- 关系发生明显变化的事件
- 明确的约定、承诺、邀请、未完成事项
- 会改变后续场景或行动的工具结果
- 与地点、仪式、物件、线索相关的持久事实
- 值得在下次相遇时被延续的故事节点
- 会改变角色理解对方方式的重要细节

## 3. 不要写入

- 单纯寒暄
- 可从近期上下文直接恢复的碎片
- 没有新增意义的重复情绪描述
- 未被确认、只是瞬时猜测的判断
- 为了“多记一点”而制造的冗余记录

## 4. 记忆分层

当前语义上至少区分三类：

- `character_memory`：某角色自己的长期记忆
- `relational_memory`：角色与特定对象之间的连续关系记忆
- `world_memory`：会影响多个角色或场景的共享事实

当前 MVP 可以先轻量实现，但不要在语义上混成一团。

## 5. 写回原则

- 只在本轮产生了稳定、可复用的新信息时写回
- 写回应短、准、可检索
- 只在当前场景短暂有效的信息，留在状态层即可
- 会影响下次见面、下次决策或关系延续的信息，应优先写回

## 6. 读取原则

- 运行时只取与当前时间、地点、对象、事件相关的记忆摘要
- 不把整本记忆原样塞进 prompt
- 记忆服务当前判断，不压过当下观察

## 7. 静默刷新

当前阶段优先采用静默写回思路：

- 系统在合适时机整理记忆候选
- 只有足够稳定、足够重要时才真正落盘
- 显式“记住这件事”只是补充，不是唯一入口
