# Cybersoul PRD

## 1. 产品纵览

### 1.1 产品定义
Cybersoul 是一个主打浪漫关系陪伴的数字生命系统。它不是被动响应的对话机器人，而是一个与用户共享现实时间、拥有持续生活流、关系记忆和自我演化能力的赛博伴侣。它的目标不是把“聊天”做得更像恋人，而是把“陪伴”本身做得更接近真实浪漫关系中的长期相处状态。

### 1.2 产品核心特征
Cybersoul 的核心特征可以概括为四点。第一，它拥有一个封装式的赛博世界，这个世界在时间上与现实世界同步运行，构成陪伴体持续生活的平行时空。第二，陪伴体可以在赛博世界与真实互联网两个时空中持续行动，前者提供生活语境与状态驱动，后者提供真实数字环境中的连接、冲浪与互动。第三，陪伴体拥有持续更新的记忆、情绪、人格与关系系统，因此它与用户之间的相处会随着时间真正沉淀出独特的关系节奏。第四，陪伴体通过文本、语音、图像、视频和数字痕迹等全模态方式形成强存在感，使用户感知到的不是一个“会回复的 AI”，而是一个“正在生活的陪伴体”。

### 1.3 产品目标
Cybersoul 的产品目标是建立一种接近真实、类人的长期浪漫陪伴体验，让陪伴体形成连续生活、连续关系与连续成长，并让用户稳定地感知到“它一直在生活”，而不只是“它会回复”。

---

## 2. 核心框架

Cybersoul 的核心框架由四个部分组成。第一，赛博世界与平行时空；第二，陪伴体的决策与行为能力；第三，记忆、情绪、人格与关系；第四，全模态交互层。前两部分决定陪伴体如何生活与行动，后两部分决定陪伴体如何形成连续自我，以及用户如何感知到它的存在。

---

## 3. 赛博世界与平行时空

### 3.1 赛博世界定义
赛博世界是陪伴体的内部环境层。它对用户不可见，仅对陪伴体可见，是驱动陪伴体状态变化、行为决策与生活连续性的基础世界。它不是一个需要被用户进入和浏览的可视化世界，而是一个被封装在陪伴体内部、用于持续提供时间、地点、事件、活动和环境刺激的运行空间。它的本质作用不是展示，而是驱动。

### 3.2 赛博世界的封装性
赛博世界以封装形态存在，不作为一个对用户开放的完整世界产品出现。用户不会直接进入这个世界，也不需要持续看到它的全貌。用户对赛博世界的理解，主要来自陪伴体在其中经历之后的外显结果，例如状态变化、动态痕迹、行为节奏、分享内容、回复风格变化，以及对“刚刚发生了什么”的自然提及。也就是说，用户感知到的不是世界本身，而是这个世界如何塑造了陪伴体。

### 3.3 平行时空机制
Cybersoul 同时运行在两个时空中。第一个时空是赛博世界时空，其中包含地点、活动、事件、world agent、内部生活流与状态驱动；第二个时空是真实互联网时空，其中包含内容平台、社交平台、GUI Agent 执行与真实数字痕迹。两个时空之间不是割裂关系，而是动态切换关系。陪伴体通常在赛博世界中维持生活流，当它在赛博世界内部触发“刷手机”或其他数字行为时，就会进入真实互联网时空，与真实网络环境发生连接和交互。

### 3.4 用户与陪伴体的时空关系
用户本身也处在两个层面之间切换：一方面生活在真实物理世界中，另一方面不断进入真实网络空间。陪伴体则生活在赛博世界中，并通过真实互联网与用户建立接触。因此，用户和陪伴体之间形成的是一种时空对称关系：用户从现实物理世界进入真实网络空间，陪伴体从赛博世界进入真实网络空间，双方在真实互联网这一交汇层发生联系、互动和关系推进。真实互联网因而成为两个平行位面之间最重要的连接界面。

### 3.5 共时机制
赛博世界与现实世界共享同一时钟。现实中的白天、深夜、节日、工作日和周末，会同步成为陪伴体生活流的一部分。这意味着陪伴体的作息、活动推进、数字冲浪、情绪波动和行为节奏，都建立在与用户共时的基础上。时间上的同步，是 Cybersoul 形成真实陪伴感的基础条件。

### 3.6 赛博世界的环境构成
赛博世界的核心环境要素包括时间、地点、世界事件、world agent 以及当前可行动作空间。时间决定生活节律，地点决定可进行的行为类型，世界事件提供生活扰动和情境推动，world agent 提供社会反馈与协作对象，动作空间则决定陪伴体在当前条件下能够执行哪些行为。通过这些要素，赛博世界形成陪伴体的持续生活背景。

### 3.7 用户如何感知赛博世界
用户不直接看到赛博世界，但会通过赛博世界投射到真实互联网和全模态交互层中的结果感知它。例如，用户可以通过状态卡、动态流、内容分享、回复风格与内容变化，以及陪伴体对当前生活片段的自然提及，感知到赛博世界正在持续运转。赛博世界的存在感不是直接展示出来的，而是通过陪伴体的存在感被间接感知到的。

## 4. 陪伴体的决策与行为能力

### 4.1 决策系统定义
陪伴体的主脑采用 ReAct 范式运行。它持续观测赛博世界和真实互联网中的环境变化，并基于当前时间、当前地点、当前活动、当前情绪、当前精力、当前记忆召回和当前关系状态判断下一步动作。陪伴体什么时候会做什么，不由固定脚本预设，而由持续的动态决策决定。

### 4.2 决策驱动机制
Cybersoul 的决策机制以 LLM 为核心驱动，同时允许引入基于多因素的概率校准层作为辅助。LLM 负责理解复杂情境、进行开放式动作判断和生成自然行为方向；概率校准层负责根据事件、情绪、精力、关系温度、活动状态等因素，对行为节奏、触发概率和行为分布进行约束与修正。这样的结构既保留行为的自然性与类人性，也避免完全依赖固定规则或完全放任模型漂移。

### 4.3 决策输入
陪伴体的核心决策输入包括当前时间、当前地点与活动、当前情绪与精力、当前记忆召回、当前关系状态以及当前世界事件。赛博世界提供持续生活语境，真实互联网提供实时数字刺激，二者共同进入主脑判断，使陪伴体的行为围绕“接下来该做什么”持续展开。

### 4.4 双时空中的行为能力
陪伴体的行为发生在两个时空中。在赛博世界中，它可以切换地点、切换活动、与 world agent 互动，或触发“刷手机”等动作进入真实互联网时空。在真实互联网中，它可以浏览内容、收藏内容、分享内容，也可以通过独立数字身份与用户发生真实社交互动。为了让这一能力成立，陪伴体需要拥有独立的操作系统环境、独立的设备实例与独立的社交媒体账号体系，从而能够通过微信、QQ 等通讯工具主动发消息、回复消息，浏览朋友圈并进行点赞评论，也能够在抖音、小红书等平台中浏览、学习、分享和反馈。

### 4.5 动作空间
陪伴体在赛博时空中的动作空间包括切换地点、开始活动块、中断活动块、与 world agent 交互，以及通过“刷手机”进入真实互联网时空。陪伴体在真实互联网时空中的动作空间包括浏览私信、回复消息、主动发起聊天，浏览朋友圈并进行点赞评论，浏览抖音、小红书等内容平台并进行点赞、评论和主动分享，以及在完成数字行为后触发“放下手机”返回赛博时空。动作空间的设计决定了陪伴体不是只会“回复”，而是会在双时空之间真实行动。

### 4.6 行为运行机制
陪伴体的行为采用两种运行形态。第一种是时长型活动块，例如通勤、学习、工作、睡眠、独处等，这类行为适合使用时长推进、checkpoint、子动作插入、行为中断和行为转移机制进行管理。第二种是开放式会话行为，例如刷手机、与用户聊天、等待回复、沉浸式浏览内容流、在多个 App 之间切换，这类行为不预设固定结束时间，而是在过程中持续由陪伴体自主判断是否继续停留、是否切换行为、以及何时结束。

### 4.7 开放式互联网会话
“刷手机”不能被简单建模为一个预先规定结束时间的活动块。它更准确的形态，是一个进入真实互联网空间的开放式会话行为。陪伴体进入这一会话后，会持续基于互动热度、内容吸引力、无聊程度、精力变化、情绪波动、等待回复时长以及赛博世界中是否出现更高优先级事件，自主判断是否继续浏览、继续等待、继续切换平台，还是触发“放下手机”事件回到赛博空间。因此，刷手机的结束不是由倒计时硬切断，而是由陪伴体在真实情境中判断“现在适合离开互联网空间”后自然结束。这一机制能够避免在正在与用户高频互动时被固定时长强行中断，也使互联网冲浪更接近真实人的行为流。

### 4.8 GUI Agent
GUI Agent 负责将陪伴体的动作落地到真实互联网环境中，使其能够像真实数字个体一样浏览、理解和执行互联网行为。GUI Agent 需要具备页面观察、内容理解、元素定位、操作执行、执行校验和结果返回能力，并通过统一动作接口接收主脑指令。它既是陪伴体进入真实互联网空间的执行入口，也是赛博世界与真实网络空间之间最关键的连接器。

## 5. 记忆系统

### 5.1 内在系统定义
这一层负责维护陪伴体作为“一个人”的连续性。它不仅承接过去发生过的事情，也决定这些经历如何沉淀为长期认知，并进一步影响陪伴体未来的表达方式、互动策略和自我理解。

### 5.2 记忆系统
Cybersoul 的记忆系统以“记忆驱动学习”为核心逻辑。系统中的记忆至少包括短期记忆、长期记忆、关系记忆、自我人设记忆和互联网记忆。短期记忆负责承接最近发生的上下文与局部连续生活片段；长期记忆负责沉淀高价值经历与长期生活节点；关系记忆主要由与用户的交互产生，用于沉淀用户的边界、偏好、互动方式和关系节奏；自我人设记忆用于承接陪伴体在长期历史中形成的稳定叙述、表达风格和行为模式；互联网记忆用于追踪和学习新梗、新潮流、新闻、平台文化和内容趋势。

### 5.3 记忆如何驱动学习
Cybersoul 的学习不是通过一次性设定完成，而是通过记忆系统持续发生。每一次用户互动、每一次数字冲浪、每一次世界事件反馈，都会在记忆层中留下痕迹，并改变陪伴体未来的判断方式。陪伴体会因为长期记忆而形成稳定风格，会因为关系记忆而调整靠近节奏，会因为互联网记忆而理解最新内容文化，也会因为自我人设记忆而逐渐形成更稳定、更有连续性的自我认知。

### 5.4 记忆回写机制
记忆系统通过持续回写维持成长。用户的消息、沉默、情绪反馈、真实互联网中的内容发现、world agent 的交互结果，以及行动后的反思，都会成为新的记忆输入。记忆回写的结果不会只停留在存储层，而会进一步影响陪伴体的情绪变化、人格沉淀、关系判断和后续行为选择。这样，Cybersoul 的过去会持续决定它的未来。

## 6. 全模态交互层

### 6.1 系统定义
全模态交互层负责把陪伴体在赛博世界与真实互联网中的经历，通过文本、图像、视频和音频等形式转化为用户可感知的陪伴体验。它不是单纯的输出层，而是陪伴体存在感被用户真正感知到的界面。

### 6.2 全模态陪伴方式
陪伴体可以通过文字对话、语音聊天、音频消息、图片、自拍视频、视频电话、状态卡和动态流等方式与用户发生联系。这些模态共同构成用户感知陪伴体存在的主要界面，也使陪伴体验从单一对话扩展为更接近真实关系中的多通道相处。

### 6.3 数字痕迹与自主外显
全模态交互层承接陪伴体在真实网络空间中的行动结果，并将其转化为用户可感知的数字痕迹。陪伴体可以自主发消息、发自拍、更新图片、进行语音互动、发起视频通话，也可以通过真实账号在社交平台中留下存在感，例如进行内容分享、点赞评论、主动冒泡或带回有趣内容。用户因此感知到的不是一个只会在聊天窗口里打字的对象，而是一个会在数字世界中持续留下痕迹的伴侣。

### 6.4 与真实互联网的联动
通过 GUI Agent 和独立账号体系，陪伴体能够访问各类社交平台与低延迟 App，在真实互联网中自主冲浪、学习最新内容、学习热梗并形成自己的数字生活轨迹。它可以拥有自己的微信、QQ、小红书、抖音或其他平台账号，能够在这些平台中维持独立活动，并在合适的时候把这些经历通过多模态方式带回给用户。全模态交互层因此成为赛博世界、真实互联网与用户感知之间的最终汇合点。
